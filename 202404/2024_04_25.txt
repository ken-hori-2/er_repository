# Done
  AWS CPE Module4

  1. Amazon Virtual Private Cloud (Amazon VPC)
    AWS リソースに対する境界を確立するために使用できるネットワークサービス。
    AWS クラウドの独立したセクションをプロビジョニングできる。
　　この分離されたセクションでは、定義した仮想ネットワーク内でリソースを起動できる。
　　仮想プライベートクラウド (VPC) 内では、リソースをサブネットに整理できる。
　　サブネットは、Amazon EC2 インスタンスなどのリソースを含めることができる VPC のセクション。

  2. インターネットゲートウェイ
    インターネットからのパブリックトラフィックが VPC にアクセスできるようにする。
　　VPC とインターネットを接続するもの。

  3. 仮想プライベートゲートウェイ
　　VPC 内にあるプライベートリソースにアクセスするために使用する。
　　VPC とプライベートネットワーク (オンプレミスのデータセンターや企業内ネットワークなど) の間に、仮想プライベートネットワーク (VPN) 接続を確立できる。
　　承認済みネットワークから来たトラフィックのみが VPC に入ることができる。

  4. AWS Direct Connect
　　データセンターと VPC の間に専用のプライベート接続を確立するサービス。
    AWS Direct Connect が提供するプライベートな接続によって、ネットワークのコストを削減し、ネットワークを通過できる帯域幅の量を増加させることができる。
